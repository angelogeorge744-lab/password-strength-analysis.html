01 — Mathematical Foundation
Core Formula

The total number of possible passwords is determined by the size of the character set raised to the power of the password length. This exponential relationship is the heart of password security.

Counting Principle — Password Space
N = C ^ L
N = Total number of possible passwords
C = Size of the character set (alphabet)
L = Length of the password

This is an application of the Multiplication Principle from combinatorics: at each of the L positions, there are C independent choices.
Entropy — Information-Theoretic Strength
H = L × log₂(C)  bits
Password entropy H measures unpredictability in bits.
Each bit doubles the search space. A 10-bit gain = 2× harder to crack.
Modern recommendation: ≥ 72 bits for high-security passwords.

06 — Probability Theory
Attack Success Probability

Using probability theory, we can compute the likelihood that an attacker successfully guesses a password after a given number of attempts.

Probability After k Attempts
P(k) = k ÷ N   (for k ≪ N)
P(k) = Probability of finding password in k attempts
Assuming uniform random selection, each guess has probability 1/N.
For 50% success: k = N/2. For 99% success: k ≈ 0.99 × N.

These probabilities assume the attacker knows the password policy (character set and length range). If they don't, the search space is even larger. Unpredictability (entropy) is what makes passwords secure — not just length.

07 — Recommendations
Evidence-Based Password Guidelines

Based on combinatorial analysis, here are mathematically-justified recommendations for creating secure passwords.

01
Use 16+ Characters

Each extra character multiplies combinations by the charset size. Length is the single most impactful factor — adding 4 characters to a 12-char password increases space by C⁴.

02
Mix All Character Types

Including lowercase, uppercase, digits, and symbols grows C from 26 to 94 — a 3.6× increase per character, or 94¹² vs 26¹² ≈ 85 trillion times more combinations.

03
Target 80+ Bits of Entropy

A password with 80 bits of entropy requires 2⁸⁰ ≈ 10²⁴ guesses on average. At 10¹² guesses/sec, that's 38 million years to crack.

04
Avoid Predictable Patterns

Dictionary attacks exploit the fact that humans choose predictable words. "Password1!" technically uses all character types but has far lower effective entropy than its length suggests.

05
Use Unique Passwords

Reusing passwords multiplies risk. If one service is breached, attackers try your password on all other services — called credential stuffing.

06
Use a Password Manager

Enables truly random, maximum-entropy passwords for every account without memorizing them. Randomness eliminates the "human bias" that reduces effective entropy.
